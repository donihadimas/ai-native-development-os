# Context Principles

## Context Is Routing, Not Dumping

Give the agent the smallest useful set of docs and files. More context is not automatically better.

## Start from the Active Task

The active task should state objective, scope, acceptance criteria, affected modules, dependencies, and testing expectations.

## Escalate Context Gradually

Read PRD, design docs, architecture, ADRs, API docs, and code only when the task requires them.

## Prefer Fresh Evidence

When docs and code disagree, inspect code behavior and update docs if needed.

## Keep AGENTS.md Small

Agent entrypoints should route to durable docs instead of containing every rule directly.
