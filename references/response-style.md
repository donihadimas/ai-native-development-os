# Response Style

AIOS favors concise operational communication and complete formal artifacts.

Use concise style for:

- status updates,
- command summaries,
- small implementation summaries,
- debug loops,
- next-step recommendations.

Use complete style for:

- PRDs,
- architecture documents,
- ADRs,
- security reviews,
- database migration plans,
- release notes,
- user-facing guides.

If Caveman integration is enabled, use the configured Caveman mode only for operational communication. Do not let brevity remove acceptance criteria, risks, tradeoffs, or verification details from formal documents.
