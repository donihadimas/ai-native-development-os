# References

References contain stable, stack-agnostic engineering principles. Skills and workflows may link to these files when durable guidance is needed.

References should describe scalable, maintainable, production-grade defaults without forcing a specific framework. They should prefer industry-standard practices: explicit contracts, secure defaults, observable behavior, safe migrations, clear ownership, backward compatibility, and testable acceptance criteria.

V2 adds backend API standards to support contract-aligned API work without choosing a specific framework.

Use `artifact-lifecycle.md` for task and plan indexes, active/archive folders, and summary-first reading rules.
