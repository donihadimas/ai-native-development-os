# Release: 0.4.0

## Scope

- Included:
  - `aios update` safe asset synchronization.
  - `aios update --accept` explicit bundled `.aios/` asset acceptance.
  - Native `task-implementation` skill coverage.
  - UI/UX design workflow, templates, prompts, docs, and starter support.
  - Dogfood `.aios/` kit and native `.agents/skills/` assets for this repository.
  - Website documentation and CLI reference updates.
- Excluded:
  - Automatic Markdown merge for locally edited `.aios/` assets.
  - npm publishing, Git tagging, and GitHub Release creation. These remain manual steps after human approval.

## User-Visible Changes

- Users can run `aios update` to bring existing projects forward with newly bundled AIOS assets and skills.
- Users can run `aios update --dry-run` to preview update actions.
- Users can run `aios update --accept` or `aios update --accept <section>` to explicitly apply bundled `.aios/` asset changes after review.
- Update output now separates line-ending-only differences from content differences.
- Generated projects and starters include stronger design workflow guidance.

## Developer-Visible Changes

- CLI update classification logic now distinguishes missing, identical, content-different, and line-ending-only kit assets.
- Release package metadata keeps homepage, repository, bugs, and author fields from the `0.3.1` release.
- Bundled CLI assets include updated skills, prompts, references, templates, workflows, starters, and project skeleton files.

## Rollback Plan

- If npm publish fails before publishing, keep `main` unchanged and fix the release branch.
- If npm publish succeeds but GitHub release/tag fails, recreate the tag or release notes from this file and `CHANGELOG.md`.
- If a critical package issue is found after publish, publish a patch release such as `0.4.1` with the fix. npm package versions cannot be overwritten.

## Post-Release Checks

- Run `npm view @donihadimas/aios version` and confirm `0.4.0`.
- Install globally or in a temp project and run:
  - `aios --help`
  - `aios init verify-project`
  - `aios validate verify-project`
  - `aios update verify-project --dry-run`
- Confirm GitHub Release links to the changelog and npm package.

## Known Risks and Follow-Up

- `development` and `main` were diverged before this preparation; merge carefully so `0.3.1` metadata is preserved.
- `aios update --accept` is intentionally overwrite-based for managed `.aios/` assets and does not merge local edits.
- Local dogfood `.aios/` assets should be reviewed before the final merge if the release should include them.
