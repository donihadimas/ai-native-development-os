# Prompt: Design API Contract

Use `skills/context-management/SKILL.md` first.

Then use `skills/api-contract-design/SKILL.md`, `references/api-standards.md`, and `templates/openapi.template.yaml` to create or update an API contract in `docs/api/`.

Read only:

- relevant PRD or feature task,
- related architecture section,
- related ADRs if API behavior depends on a technical decision,
- existing API contract files in `docs/api/`.

Do not implement frontend or backend code yet. End with contract risks, open questions, and which FE/BE tasks should link to the contract.
